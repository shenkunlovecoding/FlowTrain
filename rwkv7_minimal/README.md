The official README says the full RWKV-7 model is not just a standalone layer: it depends on carefully set initialization, parameter-specific weight decay, and learning-rate grouping. It also explicitly says the simplified training demo is slower and gives different results. This extraction therefore keeps both forms.

## Files

- `rwkv7_core.py`: pure PyTorch RWKV-7 x070 core. It keeps train-temp parameter names, formulas, value residual, GroupNorm eps, init formulas, and optimizer grouping. It replaces fused CUDA kernels, Lightning, DeepSpeed, and wandb with a minimal reference path.
- `rwkv7_full_minimal.py`: smoke-test model using the official rank formulas such as `max(32, round(...))` and default train-temp FFN ratio.
- `rwkv7_simplified_minimal.py`: smoke-test model mirroring the official simplified demo choices: small vocabulary, 2 layers, `C=32`, `HEAD_SIZE=16`, LoRA ranks fixed to 8, and digit-reversal toy data.

## Run

```bash
python rwkv7_full_minimal.py
python rwkv7_simplified_minimal.py
```

The implementation is intentionally slow. It is meant as a correctness-oriented, readable extraction. For production training, the official `train_temp/src/model.py` uses fused CUDA kernels:

- `rwkv7_clampw`
- `rwkv7_cmix_bf16_v5`
- `rwkv7_tmix_mix6_bf16_v5`
- `rwkv7_tmix_kk_pre_bf16_v5`
- `rwkv7_tmix_lnx_rkvres_xg_bf16_v1`
- `rwkv7_tmix_a_gate_bf16`
- `rwkv7_tmix_vres_gate_bf16_v1`
- `rwkv7_l2wrap_ce_bf16_v2`

## Compatibility Notes

- Parameter names are aligned with official `train_temp/src/model.py`, so a state dict generated by this module has the same high-level names: `emb`, `blocks.*.ln*`, `blocks.*.att.*`, `blocks.*.ffn.*`, `ln_out`, `head`.
- The full minimal config uses official LoRA rank formulas. The simplified config uses fixed rank 8, matching the simplified demo.
- `optimizer_groups()` keeps official grouping: `att.w0` gets `my_lr_scale=2.0`; large `.weight` tensors get weight decay; other parameters do not.
- `generate_init_weight()` ports the official initialization pass, including `ln_x.weight` layer scaling, embedding init, head orthogonal init, zero output/value projections, and reduced scale for attention key weights.
